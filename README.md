# Blogicum

## О проекте
**Blogicum** — учебный Django-проект для публикации личных записей.
На сайте можно создавать посты, распределять их по категориям, указывать местоположение, оставлять комментарии и работать с профилями пользователей.

Проект развивает базовый блог до полноценного приложения с авторизацией, страницами профилей, пагинацией, комментариями, кастомными страницами ошибок и почтовым файловым бэкендом.

## Возможности проекта
- регистрация, вход и восстановление доступа через встроенные маршруты Django;
- отдельные страницы профиля пользователя и редактирование профиля;
- публикация, редактирование и удаление постов;
- отложенные публикации по дате;
- комментарии к постам с возможностью редактирования и удаления;
- пагинация на главной странице, страницах категорий и профилей;
- вывод количества комментариев у публикаций;
- кастомные страницы ошибок 403 CSRF, 404 и 500;
- сохранение исходящих писем в директорию `sent_emails/`.

## Как запустить проект
1. Клонируйте репозиторий.
2. Перейдите в корень проекта.
3. Создайте и активируйте виртуальное окружение.
4. Установите зависимости из `requirements.txt`.

### Windows
```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

### macOS / Linux
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

## Подготовка базы данных
Из директории `blogicum/` выполните команды:

```bash
python manage.py migrate
python manage.py loaddata ../db.json
```

После загрузки фикстуры в проекте появятся демонстрационные категории, местоположения, пользователи, публикации и комментарии.

## Структура проекта
```text
blogicum/
├── blog/          # Публикации, категории, комментарии, формы и представления
├── pages/         # Статические страницы и кастомные страницы ошибок
├── users/         # Пользователи, профиль и регистрация
├── templates/     # HTML-шаблоны проекта
├── static/        # Статические файлы
├── blogicum/      # Настройки проекта
└── manage.py
```

## Полезно знать
- проект использует кастомную модель пользователя: `users.User`;
- фикстура `db.json` должна ссылаться на модель `users.user`, а не `auth.user`;
- письма сохраняются в папку `sent_emails/`, поэтому её удобно добавить в `.gitignore`.
